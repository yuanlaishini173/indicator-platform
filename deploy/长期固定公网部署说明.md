# 指标完成情况平台长期固定公网部署说明

## 推荐目标

长期固定访问建议采用“云服务器 + 域名 + HTTPS + 后台密码”。

部署完成后：

- 微信只读查看地址：`https://你的域名/share`
- 后台管理地址：`https://你的域名/admin`
- 域名根地址：`https://你的域名/` 会自动进入只读分享页

公网用户只能查看数据。上传、删除、改文件名、手工填报、报告下载等后台能力需要管理账号密码。

## 服务器建议

- 系统：Ubuntu 22.04 LTS 或 Ubuntu 24.04 LTS
- 配置：2 核 2G 起步即可
- 端口：只开放 `80` 和 `443`
- 运行目录：`/opt/indicator-platform`

## 首次部署步骤

1. 安装系统依赖

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip nginx certbot python3-certbot-nginx unzip
```

2. 上传项目到服务器

把整个 `indicator-platform` 目录上传到：

```bash
/opt/indicator-platform
```

3. 创建虚拟环境并安装依赖

```bash
cd /opt/indicator-platform
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

4. 配置后台账号密码

```bash
sudo cp deploy/indicator-platform.env.example /etc/indicator-platform.env
sudo nano /etc/indicator-platform.env
```

把 `INDICATOR_ADMIN_PASSWORD` 改成强密码。

5. 安装系统服务

```bash
sudo cp deploy/indicator-platform.service /etc/systemd/system/indicator-platform.service
sudo chown -R www-data:www-data /opt/indicator-platform
sudo systemctl daemon-reload
sudo systemctl enable indicator-platform
sudo systemctl start indicator-platform
sudo systemctl status indicator-platform
```

6. 配置 Nginx

```bash
sudo cp deploy/nginx-indicator-platform.conf /etc/nginx/sites-available/indicator-platform
sudo nano /etc/nginx/sites-available/indicator-platform
```

把 `your-domain.example.com` 改成你的正式域名。

```bash
sudo ln -s /etc/nginx/sites-available/indicator-platform /etc/nginx/sites-enabled/indicator-platform
sudo nginx -t
sudo systemctl reload nginx
```

7. 绑定 HTTPS

域名解析到服务器公网 IP 后执行：

```bash
sudo certbot --nginx -d 你的域名
```

完成后，微信访问：

```text
https://你的域名/share
```

## 数据备份

需要定期备份以下目录：

```text
/opt/indicator-platform/data
/opt/indicator-platform/uploads
```

建议每天自动备份一次。最重要的是：

```text
/opt/indicator-platform/data/platform.db
```

## 管理方式

外部用户：

```text
https://你的域名/share
```

管理员：

```text
https://你的域名/admin
```

浏览器会弹出账号密码框：

- 账号：`INDICATOR_ADMIN_USER`
- 密码：`INDICATOR_ADMIN_PASSWORD`

## 重要安全说明

- 不要把 `/opt/indicator-platform/data` 和 `/opt/indicator-platform/uploads` 作为静态目录暴露出去。
- 不要开放数据库端口。
- 后台密码必须使用强密码。
- 服务器安全组只开放 `80`、`443`，SSH 端口只允许管理员 IP 更好。
- 如果要给很多人长期访问，优先使用单位认可的云账号或企业云环境。
